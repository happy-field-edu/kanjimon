# 🎮 カンジモン 3D化 引き継ぎドキュメント（for Fable）

このドキュメントは、既存の2D版Webゲーム「カンジモン（漢字島クイズ）」を **3D化** して作り直すための仕様書です。
現行の2D版は完成・公開済みで、**ゲーム性・データ構造・チューニング値・アセットはすべてこの2D版を「正」** とします。

- **現行2D版（プレイ可）**: https://happy-field-edu.github.io/kanjimon/

## 1. ゲーム概要

- **タイトル**: カンジモン（漢字島クイズ）
- **ターゲット**: 小学2年生（8歳）
- **ジャンル**: 漢字学習 × モンスター収集RPG
- **狙い**: 「正しい漢字」と「子どもがよく間違える形の漢字」を **画像4択** で見分けさせ、手書き認識の精度問題を回避。間違いはこっそり記録し、後で「ボス戦」として復習させることで“やり直し”を熱狂イベントに昇華する。
- **トーン**: ファンタジーRPG風、明るくワクワク。ボタンはプニッと凹む。横画面前提。
- **文章はすべて ひらがな中心**（8歳が読める表記）。

## 2. ゲーム全体フロー

```
タイトル画面
 ├─ CSV URL入力（先生用・任意）／サンプル問題であそぶ
 ├─ 📖 ずかん を見る（未プレイでも閲覧可）
 └─ 🏅 バッジ を見る
        ↓ スタート
メインのクイズループ
 ├─ 木の看板に「問題文」（例：「きいろ」の「き」）
 ├─ 4体のスライムが看板に「選択肢の漢字画像」を持って並ぶ（正解1＋誤字3）
 ├─ タイムボーナスゲージ（10秒で減少）
 ├─ タップで解答
 │   ├─ 正解 → エフェクト＋効果音＋EXP（残り時間ボーナス）＋コンボ加算
 │   │        └─ 一定確率でタマゴドロップ → ふらん器へ
 │   └─ 不正解 → 正解を見せる＋コンボリセット＋「ニガテリスト」に記録
 ├─ 正解のたび ふらん器のタマゴが「あとN問」カウントダウン → 0で孵化
 │   └─ 孵化 → カンジモン誕生（初発見ならNEWバッジ）／
 │             同種を規定数集めていたら【進化】演出（紙吹雪）
 └─ 10問クリアごと & ニガテが3個以上 → 【ボス襲来】自動発動
        ↓
ボス戦（専用背景シーンに切替）
 ├─ WARNING演出 → ボス出現（3種ランダム：見た目・背景・HP・名前が変化）
 ├─ ニガテ漢字から優先出題
 ├─ 正解 → ボスにダメージ＋そのニガテをリストから削除
 ├─ 不正解 → ボスの攻撃（画面赤シェイク＋次問のタイム短縮ペナルティ）
 └─ HP0で撃破 → 紙吹雪＋確定で【超レアタマゴ】ドロップ → 通常ループへ復帰
```

## 3. コア機能の仕様（3Dでも必ず再現）

### 3-1. 問題のCSV読み込み（先生が編集）
- Googleスプレッドシートの「ウェブに公開（CSV）」URLを `fetch` で読む。
- **CSVカラム**: `問題文, 正解画像URL, 間違い画像URL1, 間違い画像URL2, 間違い画像URL3`
- 1行目ヘッダーは自動スキップ。読み込み失敗時はサンプル問題にフォールバック。
- URLは `localStorage` に保存（次回自動入力）。
- 3D版でも「先生がスプレッドシートで問題追加」できる仕組みは維持すること。

### 3-2. 4択クイズ
- 各問：正解1＋誤字3をシャッフルして4体のスライムに割り当て。
- スライムは4色（緑・青・ピンク・黄）。それぞれ看板を持ち、看板の白紙に選択肢漢字を表示。
- タップ演出：プニッと凹む／正解はジャンプ＋発光／不正解はシェイク＋グレー化。

### 3-3. タイム・コンボ・EXP・レベル
- タイムゲージ：`questionTime=10秒`。緑→黄→赤に変化。残り割合に応じてEXPボーナス（最大 `timeExpMax=50`）。
- コンボ：連続正解で加算。`fireCombo=3` 以上で炎演出。不正解で0。
- EXP：正解で `baseExp=50` ＋ タイムボーナス。`LEVEL_EXP=300` ごとにレベルアップ。

### 3-4. タマゴのドロップ＆孵化（ふらん器）
- 正解時、ドロップ率 = `min(maxDropRate=0.35, baseDropRate=0.08 + combo × comboDropUp=0.02)`。コンボが高いほどレア寄り抽選。
- ドロップしたタマゴはふらん器へ（1個セット＋待ち行列）。
- タマゴは種類ごとに「孵化に必要な正解数」があり、正解のたび `あとN問` カウントダウン。0で孵化演出→カンジモン誕生。
- **卵種と孵化必要数（need）**: しろ9／あか12／あお15／にじ21／レア6。

### 3-5. ずかん（図鑑）
- 全カンジモンを一覧。**未入手は「？？？」シルエット**。「X / N」コンプリート率＋ゲージ。
- 捕獲数バッジ（×2以上で金枠）、進化段階（しんか／さいしゅうしんか）、「しんかまで あとN」ヒント表示。
- タイトル・ゲーム中の両方から開ける。開いている間はタイマー一時停止。

### 3-6. 2段階進化
- **同じ種を集めた累計数で進化**（3D版でも捕獲数ベース）。
  - コモン系統：3匹で第1進化 → 6匹で最終進化
  - にじ系統：2匹で第1進化 → 4匹で最終進化
- 進化ラインは「**同じ生き物のまま大きく・強く**」なる（別の生き物にしない）。
- 孵化直後にしきい値を跨いだら、その場で「◯◯に しんかした！」演出＋**画面いっぱいの星＆虹色の紙吹雪**（最終進化は特大）。

### 3-7. NEWバッジ・称号バッジ
- 初発見のカンジモンには孵化時「NEW! ずかんに とうろく！」バッジ。
- 称号バッジ（下記10種）は条件達成で自動付与＋「バッジ ゲット！」ポップアップ。未獲得は🔒＋条件ヒント表示。

### 3-8. ボス襲来イベント（本作の目玉・復習の昇華）
- **発動条件**: 通常クイズを `bossEvery=10` 問クリアするタイミングで、ニガテリストが `bossNeed=3` 個以上あれば自動発動。デバッグ用の強制発動ボタンも用意。
- **演出**: 赤黒フラッシュの「WARNING!! ◯◯が あらわれた！」→ 背景が **ボス専用シーン** に切替。
- **バリエーション**: `BOSSES` からランダム抽選（見た目・背景・HP・名前が変わる）。現状3種（ほのおオニ／どくオニ／こおりオニ）。
- **ルール**: ニガテから優先出題。正解でHP-1＆そのニガテを削除。不正解はボスの攻撃（赤シェイク＋次問タイム `bossPenalty=0.45` スタート）。
- **撃破報酬**: HP0で **確定の超レアタマゴ（type:"rare"）** ドロップ＋紙吹雪。背景は平原に復帰。
- **ニガテリスト**: 通常クイズで不正解になった問題文を `localStorage` に蓄積。ボスで正解すると削除。

### 3-9. セーブ
- `localStorage` キー `kanjimon_save_v1` に、EXP/レベル/ふらん器の卵/待ち行列/コレクション/ニガテリスト/統計(stats)/バッジ(badges) を保存。
- CSV URLは別キー `kanjimon_csv_url`。

## 5. カンジモン全ロースター（35体：通常33＋レア2）

各系統 = **1種類の生き物が3段階で進化**。しきい値：コモン=3/6匹、にじ=2/4匹。
（表記：`初期 → 第1進化 → 最終進化`、括弧はファイル名 / モチーフ）

### ⚪ しろタマゴ（もじ・みどりテーマ）
- モジスラ(mojisura) → モジドラ(mojidora) → ダイモジオウ(daimojiou)  … スライム→ドラゴン
- フデコロ(fudekoro) → フデチョウ(fudechou) → マイフデ(maifude)  … いもむし→蝶→孔雀
- カキッパ(kakippa) → カキドリ(kakidori) → ショウオウ(shouou)  … ヒヨコ→鳥→フクロウ

### 🔴 あかタマゴ（ほのおテーマ）
- ヒノカキ(hinokaki) → カエンコ(kaenko) → ゴウエンロウ(gouenrou)  … キツネ→オオカミ→トラ
- モエルン(moerun) → モエグマ(moeguma) → バクエングマ(bakuenmanmo.png)  … ハムスター→クマ
- アカジシ(akajishi) → グレンオウ(gurenou) → エンテイオウ(enteiryu.png)  … ライオンで統一

### 🔵 あおタマゴ（みず・こおりテーマ）
- スイスイ(suisui) → オオシオ(ooshio) → カイテイオウ(kaiteiou)  … イルカ→クジラ
- アオガメ(aogame) → オオガメ(oogame) → ゲンブオウ(genbuou)  … カメで統一
- コオリン(koorin) → フブキペン(fubukipen) → コオリオウ(kooriou)  … ペンギンで統一

### 🌈 にじタマゴ（でんせつ・レア／しきい値2・4）
- ニジマル(nijimaru) → セイナウマ(seinauma) → セイテンバ(seitenba)  … ユニコーンで統一
- ヒカリュウ(hikaryu) → コウテイリュウ(kouteiryu) → シンリュウ(shinryu)  … ドラゴンで統一

### ✨ レア（ボス撃破のレアタマゴ限定・進化なし）
- オウゴンマル(ougonmaru) … 黄金のスライム
- ヒカリオウ(hikariou) … 光の王

> ※ 卵種→出現プール：しろ卵はしろ系3種からランダム…の対応。孵化時にどの初期種になるか抽選し、以降は累計捕獲数で進化段階を導出（同名がキー）。

## 6. アセット一覧（images/ に約35体分＋ボス＋背景＋スライム）

- カンジモン：上記35体の透過PNG（512px前後、1:1、透過背景）
- ボス：`boss_01/02/03.png`（本体スプライト）＋ `boss_bg / boss_bg_02 / boss_bg_03.png`（全画面背景シーン）
- 選択肢スライム：`slime_card.png`(緑) `slime_blue.png` `slime_pink.png` `slime_yellow.png`（看板を持つ・白紙は空白）
- タマゴはSVGで動的生成（種類ごとに色/ヒビ段階）。3Dでは卵モデル＋マテリアル色替えで再現推奨。
- **3D化する場合**：これらの2Dイラストを「参考デザイン」として3Dモデル化するか、当面は板ポリ（ビルボード）としてそのまま3D空間に立てるかは選択可。まずはビルボード方式で全機能移植→順次3Dモデル化、が低リスク。

## 7. 3D化の提案（技術・演出）

- **推奨技術**: Three.js（軽量・Web・スマホ対応）。物理は不要（演出はTweenで十分）。UIはHTML/CSSオーバーレイ（看板・ゲージ・図鑑・バッジ）と3Dシーンの併用が現実的。
- **シーン構成案**:
  - 通常：ゆるやかな「漢字島」の3Dステージ。手前に4体のスライム（3Dモデル or ビルボード）、中央奥に木の看板（3Dパネル）。カメラは固定気味の見下ろし〜正面。
  - 孵化/進化：カメラ寄り＋パーティクル（星・紙吹雪）を3Dで。最終進化はカメラ回り込み＋発光。
  - ボス戦：背景をボスの3Dアリーナに切替（スカイボックス or 背景ドーム）。ボスは中央に大きく。被弾でカメラシェイク＋赤フラッシュ。
- **演出の3D化の勘所**（現行の“気持ちよさ”を必ず継承）:
  - タップ時のプニッ、正解ジャンプ＋発光、コンボ炎、タマゴが放物線で飛ぶ、孵化のヒビ→誕生、進化の白光→変身、WARNING赤黒フラッシュ、撃破の紙吹雪。
- **維持必須**: CSV読み込み／ニガテ記録／ボス発動条件／セーブ形式（できれば `kanjimon_save_v1` 互換だと2D版の進捗を引き継げる）。

## 8. 引き継ぎの優先順位（Fableへ）

1. まず **クイズのコアループ**（CSV読み込み→4択→タイム/コンボ/EXP）を3Dで動かす。
2. **タマゴ→孵化→コレクション→図鑑** を移植（収集の中毒性がこのゲームの背骨）。
3. **2段階進化＋演出**（達成感の山場）。
4. **ボス襲来（ニガテ復習）＋レア報酬**（本作の目玉）。
5. **バッジ・仕上げ演出**。
- 各ステップで **現行2D版と同じ数値・タイミング** になっているか確認しながら進めること。
- 8歳が迷わないUI（大きいボタン・ひらがな・横画面）を死守。

### 補足：現行2D版の主なファイル/構造
- `index.html` 単一ファイル。主な定数：`CONFIG` / `EGG_TYPES` / `MON_POOLS`（系統・evo・evo2）/ `MON_IMAGES`（名前→画像パス）/ `BOSSES` / `BADGES` / `SLIME_PICS`。
